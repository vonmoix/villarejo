Accessibility & Reliability critical

critical solved

No prefers-reduced-motion support

The site runs 96 animated SVG paths plus framer-motion scroll-spy animations on every page load. Users who enable reduced motion (vestibular disorders, epilepsy) have no escape. This is a WCAG 2.3.3 failure.

// components/ui/background-paths.tsx — no reduced-motion guard // components/sections/experience.tsx — scroll-spy animation runs unconditionally // app/globals.css — missing @media (prefers-reduced-motion: reduce) block

Add @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } } to globals.css. In FloatingPaths, check window.matchMedia('(prefers-reduced-motion: reduce)').matches before mounting the animation.

globals.css — full @media (prefers-reduced-motion: reduce) block added. FloatingPaths now uses framer-motion useReducedMotion() hook; isAnimating = active && !shouldReduce stops all 96 paths when user opts out.

critical solved

Dead footer links (Privacy, Terms, DMCA)

The footer renders three navigation links that resolve to /privacy, /terms, and /dmca — none of which exist. Any visitor clicking them hits a 404, which looks unprofessional on a portfolio site intended to signal engineering rigour.

// components/sections/footer.tsx line 7-14 ["Privacy", "Terms", "DMCA"].map(item => ( <a href={`/${item.toLowerCase()}`}> // → 404 on click ))

Either create stub pages for those routes, or remove the links entirely. A simpler footer (just copyright + LinkedIn) is more honest than legal links that go nowhere.

Footer simplified to copyright + dynamic year + LinkedIn link only. Dead routes removed.

critical solved

No visible keyboard focus styles

The globals.css and Tailwind config contain no custom focus-visible styles. All interactive elements (nav links, buttons, email link, tech-stack hover targets) rely on the browser default, which is suppressed by -webkit-font-smoothing in some contexts. Keyboard-only users cannot tell where focus is.

// app/globals.css — no :focus-visible rules // components/nav.tsx — anchor tags lack explicit focus styles // components/sections/contact.tsx — email anchor lacks focus style

Add a global rule: :focus-visible { outline: 2px solid var(--accent); outline-offset: 3px; border-radius: 2px; }. Also add outline: none to :focus:not(:focus-visible) to suppress mouse-click outlines.

globals.css — :focus-visible { outline: 2px solid var(--accent); outline-offset: 3px; border-radius: 2px; } added globally. Mouse-click outlines suppressed via :focus:not(:focus-visible) { outline: none; }.

Design & UX warning

warning solved

Color tokens in hex instead of OKLCH

The entire palette is defined in hex in globals.css. OKLCH gives perceptually uniform steps — equal lightness increments look equal, which hex/HSL cannot guarantee. At the current scale (8 tokens) the difference is subtle, but it matters when extending the palette (e.g., adding hover states, disabled states, or a light-mode variant).

// app/globals.css --bg: #111009; /* OKLCH equivalent: oklch(0.12 0.008 62) */ --accent: #f59e0b; /* oklch(0.72 0.17 56) */

Migrate tokens to OKLCH. Modern browsers support it natively. The conversion is mechanical and improves palette extensibility significantly.

All 10 palette tokens migrated to OKLCH in globals.css. Brown-tinted neutrals now use perceptually uniform steps.

warning solved

FloatingPaths performance: 96 animated SVG paths on every load

Two FloatingPaths components each render 48 <motion.path> elements with continuous pathOffset animations (22–41s cycles). On mid-range devices this can cause sustained GPU load even when the hero is not in view, because the animation doesn't stop on scroll-away.

// components/ui/background-paths.tsx // 48 paths × 2 instances = 96 permanently animating SVG paths // pathOffset: [0, 1, 0] with repeat: Infinity — never pauses

Add an IntersectionObserver (or framer-motion's useInView) to the outer container and pause the animations when the hero scrolls out of view. Also guard behind prefers-reduced-motion.

HeroPaths now uses useInView({ once: false }) on the section ref. FloatingPaths receives active prop — when hero leaves viewport, framer-motion cancels the Infinity loops and transitions paths to static values (1.2s easeOut). Also guarded by useReducedMotion().

warning solved

13 of the 16 tech logos load from cdn.simpleicons.org. If that CDN is slow or blocked (corporate firewalls, China, some VPNs), the entire icons row renders as empty boxes with no text-only fallback. The portfolio would look broken to some recruiters.

// components/sections/tech-stack.tsx src: `${SI}/googlecloud/f59e0b` // external request, no fallback src: `${SI}/docker/f59e0b` // 13 requests total

Add onError to each <img> that shows the tech name as a styled text badge instead. Or better: move all logos to inline SVG via the simple-icons package (already installed).

onError handler added to all CDN img elements — renders styled mono text badge on failure. AWS, Azure, Microsoft 365 moved to inline SVG paths from simple-icons npm package (fill="currentColor" + CSS color: var(--accent)) to avoid CDN trademark removals.

warning solved

scroll-behavior: smooth overrides user preference

Setting scroll-behavior: smooth globally in CSS forces smooth scrolling for all anchor navigation regardless of the user's OS "reduce motion" setting. Browsers are supposed to honour that preference, but scroll-behavior: smooth on html bypasses it in some engines.

// app/globals.css line 21 html { scroll-behavior: smooth; }

Replace with @media (prefers-reduced-motion: no-preference) { html { scroll-behavior: smooth; } } so users who opt out get instant jumps.

globals.css — scroll-behavior: smooth moved inside @media (prefers-reduced-motion: no-preference) block.
