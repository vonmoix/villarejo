01

First Impression

What lands in the first 2 seconds — emotional reaction, purpose clarity, and primary attention anchor.

Immediate emotional read Positive

The hero is arresting. The oversized Bricolage Grotesque name at clamp(4rem, 11vw, 9.5rem) with animated amber SVG paths in the background creates a confident, premium first impression. The grain overlay at opacity: 0.032 adds material texture without being heavy. The amber accent colour (#F59E0B) reads as authoritative and distinctive — unusual enough to stand out among IT-sector portfolios that default to blue.

The purpose is clear within 2 seconds: this is a named person, in a senior leadership role, and they want you to scroll or get in touch. The scroll indicator (magic-mouse oval with animated dot) is a nice functional cue that doesn't crowd the composition.

Subtitle weight relative to name P3 — Minor

The subtitle "IT Director & Enterprise Engineering Leader" renders at clamp(1rem, 2.2vw, 1.45rem) — roughly 5–6× smaller than the name. On wide viewports the contrast in scale is extreme. The role declaration could sit at a slightly larger minimum to ensure legibility on mobile (where 1rem at 15px can feel thin below a 64px name).

Recommendation

Change the min clamp value from 1rem to 1.125rem in background-paths.tsx:149. The visual contrast remains dramatic but the subtitle becomes more readable on small phones.

02

Usability

Can the user accomplish their goal? Navigation reach, interactive affordances, and task completion.

Navigation doesn't link to all sections P1 — High

The nav links array in nav.tsx:7-12 only exposes four anchors: #experience, #education, #skills, #about. Two fully-implemented sections — Tech Stack (id="tech") and Certifications (id="certifications") — are unreachable from the navigation.

A recruiter or hiring manager scanning the site via nav clicks will never see the tech logos or professional credentials. These are high-signal sections for an IT Director audience.

Recommendation

Add both anchors to the nav links array, or group them under a "More" dropdown to keep the header uncluttered. At minimum, add #certifications since credentials are P1 content for this persona.

// nav.tsx:7
const links = [
  { href: "#experience",    label: "Experience"    },
  { href: "#education",     label: "Education"     },
  { href: "#skills",        label: "Skills"        },
  { href: "#tech",          label: "Tech Stack"    }, // ← add
  { href: "#certifications", label: "Credentials"   }, // ← add
  { href: "#about",         label: "About"         },
];
Duplicate CTA — email link + "Email Directly" button P2 — Moderate

In contact.tsx, the email address is presented as a large clickable link (line 34-41) and then repeated as the primary button "Email Directly" (line 44-46) — both pointing to the same mailto:. This creates redundancy and slightly dilutes the clarity of the CTA hierarchy.

Recommendation

Keep the animated underline email link as the visual statement, and change the primary button to something additive — e.g. "Download CV" (if a PDF is available) or remove the duplicate button entirely and keep only the LinkedIn button as the secondary action.

Timeline mobile layout untested at narrow widths P2 — Moderate

The Experience timeline uses a fixed three-column grid in experience.tsx:178: gridTemplateColumns: "1fr 44px 1fr". No responsive override exists. At 360px viewport width, each content column is only ~158px wide with 6px padding — enough for a role title but barely for the bullet text (max-width is 52ch, which renders as overflow-triggering word-wrap).

The alternating left/right pattern also means half the content is in an unexpected position on first glance for a mobile user who's used to single-column timelines.

Recommendation

Below md, collapse to a single-column timeline: the dot moves to the left edge, all cards stack in one column. This is a known responsive pattern for alternating timelines.

/* At sm breakpoint */
@media (max-width: 768px) {
  .timeline-item {
    grid-template-columns: 20px 1fr; /* dot | content */
  }
}
Mobile menu lacks visual close affordance P3 — Minor

The mobile hamburger correctly transforms to an ✕ on open (via CSS rotation in nav.tsx:68-70), but the ✕ is 20×20px at muted colour. At rgba-based muted the contrast may be marginal. Additionally, there's no label change on the button from "Toggle navigation" when open — a screenreader would not know the current state is "close".

Recommendation

Add a dynamic aria-label: open ? "Close navigation" : "Open navigation". Also consider bumping the ✕ size to 22×22px or using a slightly brighter colour in the open state.

03

Visual Hierarchy

Is there a clear reading order? Are the right elements emphasised? Does whitespace work?

Reading flow across sections is excellent Positive

Every section follows the same eyebrow → H2 → body text → content pattern. The eyebrow uses font-mono at 0.6875rem in text-accent-hi — it reads as a category label, not competing with the headline. The H2 uses clamp sizing that scales beautifully between mobile and desktop.

The amber rule (w-11 h-0.5) in the hero creates a crisp visual anchor between name and subtitle — a small detail with high compositional impact.

Hero name at 9.5rem risks ultra-wide overflow intent P3 — Minor

At viewport widths above ~1400px, clamp(4rem, 11vw, 9.5rem) hits the 9.5rem ceiling — 152px. "VILLAREJO" at this size wraps to two lines on viewports below ~900px wide but the clamp minimum (4rem = 64px) kicks in before that. The transition is handled, but on a 1920px display the name is capped at 152px while the available canvas is 1100px — a large whitespace gap appears left and right of the name block which reads as unintentional.

Recommendation

Increase the max clamp value to 10.5rem or 11rem so the name fills more of the available width on large monitors. Alternatively, set a max-width on the hero content block to create deliberate left-side framing.

About section image aspect ratio 21:8 is landscape-extreme P3 — Minor

The panoramic image in about.tsx:10-24 uses a 21 / 8 aspect ratio — approximately 2.6:1. On mobile where the image is 100vw, this renders as only ~135px tall on a 360px screen. The vignette gradient helps it blend, but the image contributes very little visual content at that height and may look unintentional.

Recommendation

Use a responsive aspect ratio — e.g. 21/8 on desktop, 16/9 on mobile — via a Tailwind aspect-[16/9] md:aspect-[21/8] approach.

Skills tag grid could benefit from visual grouping emphasis P3 — Minor

The five skill groups in skills.tsx use a 3-column grid with equal visual weight. The "Emerging Technologies" group (AI, agentic tools) is arguably the highest-signal differentiator for this profile — but there's no visual way to distinguish its importance from "Leadership & Operations".

Recommendation

Consider adding a subtle border-accent/20 or a faint amber glow on the "Emerging Technologies" card to signal it as a current-focus area. Or reorder the groups so it appears first or last (currently it's 5th).

Consistency

Design system adherence — spacing, colour, typography, and behavioural patterns across sections.

Section bottom margin inconsistency across H2 blocks P2 — Moderate

After the section subtitle paragraph (the descriptive line beneath each H2), margin-bottom is inconsistent:

Recommendation

Standardise to mb-16 (64px) after the subtitle paragraph in all sections that have one. For sections without a subtitle, standardise H2 bottom margin to mb-14 or mb-16.

Section padding rhythm is consistent and well-executed Positive

Every section uses py-24 md:py-32 or py-28 md:py-40 — there are only two tiers, which creates a predictable breathing rhythm. Background alternation (sections with bg-surface vs transparent) is used effectively to separate content without hard borders.

Inline hex colours alongside CSS tokens P3 — Minor

Most colour usage correctly references Tailwind tokens (e.g. text-accent, border-[var(--border-subtle)]), but some spots use raw hex directly:

  • stroke="#F59E0B" in background-paths.tsx:32 — should be stroke="var(--accent)"
  • backgroundColor: "#F59E0B" in experience.tsx:200 — framer-motion animate prop
  • const AMBER = "#f59e0b" in tech-stack.tsx:9 — used for CDN URL construction (acceptable here)
Recommendation

Use var(--accent) wherever CSS variables are supported. For Framer Motion's animate object, define the hex once as a constant from a shared tokens file to keep a single source of truth.

05

Accessibility

WCAG 2.1 AA checklist — contrast, keyboard navigation, ARIA, reduced motion.

No skip navigation link P0 — Critical

WCAG 2.4.1 (Bypass Blocks, Level A) requires a mechanism to skip repeated navigation. There is no <a href="#main-content">Skip to content</a> link at the top of the document. Keyboard-only users must tab through all 6 nav links on every page load before reaching any content.

Fix — add to layout.tsx
// app/layout.tsx — inside <body>, before {children}
<a
  href="#main-content"
  className="sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4
             focus:z-[9999] focus:bg-accent focus:text-bg focus:px-4
             focus:py-2 focus:rounded-[3px] focus:font-medium"
>
  Skip to main content
</a>
<main id="main-content">{children}</main>
Mobile menu overlay missing ARIA role and focus trap P1 — High

The full-screen mobile menu in nav.tsx:77-96 is a <motion.div> with no role="dialog", no aria-modal="true", and no focus trap. When opened, keyboard focus continues flowing into content behind the overlay. A screenreader user pressing Tab will navigate into invisible page content.

Recommendation

Add role="dialog" aria-modal="true" aria-label="Navigation menu" to the overlay div. Implement a focus trap (e.g. focus-trap-react or manual tabIndex management). Move focus to the first menu item on open, and return focus to the burger button on close.

Colour contrast — small text at text-subtle P1 — High

--subtle: #5c5651 on --bg: #0f0e0c yields a contrast ratio of approximately 3.8:1. WCAG AA requires 4.5:1 for normal text (<18px / not bold). This token is used for:

  • • Certification years (certifications.tsx:32) — font-mono text-[0.6875rem]
  • • Tech stack labels (tech-stack.tsx:109) — font-mono text-[0.5625rem]
  • • Education location (education.tsx:49) — text-[0.8125rem]
  • • About language level (about.tsx:51) — font-mono text-[0.6875rem]

All of these are below 14px (the threshold where Bold at 4.5:1 relaxes to 3:1 for large text). They fail WCAG AA.

Recommendation

Lighten --subtle to approximately #706b65 (contrast ≈ 4.6:1 on #0f0e0c) or bump these specific instances to use text-muted (#9b9488, ratio ≈ 7.2:1). The visual change is minimal — one step up the grey scale.

useReducedMotion correctly respected in hero Positive

background-paths.tsx:6-8 checks useReducedMotion() and disables the animated SVG paths when the OS prefers reduced motion. This is excellent WCAG 2.3.3 (Animation from Interactions) compliance. However, the per-letter hero text animation (48 motion.span elements) and all other useInView-triggered animations across sections do not check this preference.

Recommendation

Add a global reduced-motion variant. The cleanest approach: set initial and animate to the same value when prefersReducedMotion is true, or use the CSS @media (prefers-reduced-motion: reduce) approach alongside Framer's LazyMotion.

LinkedIn button opens in new tab — no announcement P2 — Moderate

The LinkedIn anchor in contact.tsx:48-55 has target="_blank" but no aria-label indicating the new-tab behaviour. Screenreader users are typically unaware a new tab opened. The ↗ glyph is decorative and has no accessible text equivalent.

Recommendation

Add aria-label="LinkedIn profile (opens in new tab)" to the link. This satisfies WCAG 2.4.4 (Link Purpose) and communicates context to screenreader users.

06

Motion Design

Animation quality, purposefulness, and performance characteristics.

  • ✓ Hero letter-by-letter entrance. 48 motion.span elements with staggered spring animation. Delay offset of 0.028s per letter creates a clean cascade without feeling slow. Spring params (stiffness 130, damping 22) feel natural.
  • ✓ Animated SVG paths in hero background. 48 paths × 2 directions, GPU-composited. Paused via useInView(once: false) when hero scrolls out — excellent GPU management. Each path gets a unique duration (22s + id×0.4s) so they desync naturally.
  • ✓ Timeline scroll-spy active state. The spring-animated scale + amber dot glow is satisfying. Using 42% of viewport height as the target creates a natural focal point that feels intentional.
  • ✓ Staggered section reveals. Consistent opacity: 0 → 1, y: 14 → 0 entrance with once: true scroll trigger. Delays are short (0.045–0.1s per item) — doesn't feel artificially slow.
  • ⚠ 48 animated paths in hero run indefinitely. Even with the heroInView pause, on low-end devices 48 concurrent SVG path animations may cause jank. Consider reducing to 32 paths or using will-change: opacity on the SVG container (background-paths.tsx:24).
  • ⚠ Per-letter animation doesn't respect prefers-reduced-motion. The 96 motion.span elements (first + last name) animate regardless of OS accessibility settings. See Accessibility section for fix.
  • ✓ Scroll indicator is elegantly minimal. The magic-mouse oval with amber bouncing dot (background-paths.tsx:167-197) uses a custom [0.45, 0, 0.55, 1] easing with repeatDelay: 0.3. Subtle but communicative.
  • ✓ Mobile menu transition. opacity: 0 → 1, y: -6 → 0 at 0.22s with a Material Design easing curve [0.4, 0, 0.2, 1]. Fast enough to feel snappy, slow enough to feel polished.
Hero grain overlay is position:fixed from within a section component P2 — Moderate

The film grain overlay in background-paths.tsx:101-109 is a position:fixed, z-50 div rendered inside the HeroPaths component. This means it's mounted when the Hero component mounts and unmounted when it unmounts (though with Next.js it's persistent throughout the SPA session). The architectural concern: it applies globally across all scroll positions, but it's owned by the Hero — a future developer might remove the Hero and accidentally remove the grain from the whole page.

Recommendation

Move the grain overlay to app/layout.tsx so it's explicitly global. It's a 1-line move and removes the implicit coupling between Hero and the page-level texture.

07

What Works Well

Genuine strengths worth preserving through any future iteration.

Typography system

Bricolage Grotesque / Hanken Grotesk / Fira Code is an unusual and cohesive triple stack. Display, body, and mono each serve distinct roles with no overlap. The tight letter-spacing on Bricolage (-0.035em at hero scale) is expertly calibrated.

Amber accent colour

#F59E0B is warm, high-contrast, and distinctive. Used precisely: accent text, dots, borders — never as a fill on large surfaces. The glow ring on the active timeline dot (rgba(245,158,11,0.18) at 5px spread) is elegant.

Timeline scroll-spy

The active highlight system using closest-center-to-42%-viewport is genuinely delightful UX. It creates a reading assistant effect — the site responds to your scroll position and guides attention without being intrusive.

Token-based colour system

The CSS variable approach with semantic names (--ink, --muted, --subtle, --accent, --bg, --surface) is clean and maintainable. Warm-tinted darks (#0f0e0c vs a cold #111111) give the palette character.

Monospace label pattern

Using Fira Code for dates, labels, and section eyebrows creates a professional, systematic feel — appropriate for an engineering/IT persona. The tracking-[0.12em] + uppercase treatment at 0.6875rem is precisely right.

Tech stack fallback UX

tech-stack.tsx gracefully handles failed CDN logo loads with an onError handler that falls back to a mono text abbreviation. This is production-quality resilience — most portfolios just show broken images.

Imperial MBA distinction badge

The with Distinction tag in education.tsx:51 uses an amber border/text treatment that draws exactly the right amount of attention — notable without shouting. A small but high-impact design decision.

Hero GPU management

Pausing the 48-path SVG animation when the hero scrolls out of view (useInView(once: false)) shows performance awareness beyond what most portfolio sites implement. Extends battery life on mobile.

08

Priority Recommendations

Ordered by impact and effort. P0 and P1 items should be resolved before the site is widely shared.

1

Add skip navigation link

P0

WCAG Level A requirement. Add a visually hidden, focusable skip link at the top of layout.tsx that targets #main-content. 5-minute fix, maximum accessibility impact. An IT Director recruiting for a team is likely to screen their own site for accessibility signals.

2

Fix subtle colour contrast failure

P1

Lighten --subtle from #5c5651 to #706b65 in your globals.css. This brings the token to a 4.6:1 ratio — passing WCAG AA — while preserving the warm grey aesthetic. Affects certification years, tech labels, and location text.

3

Add Certifications and Tech Stack to nav

P1

Two fully-implemented sections are invisible to visitors who use the nav for orientation — which is most desktop users. Credentials and tech stack are exactly what a CTO or board member scans for in an IT Director portfolio. Add both anchors to nav.tsx. 10-minute change.

4

ARIA + focus trap for mobile menu

P1

Add role="dialog", aria-modal="true", and a focus trap to the full-screen mobile menu overlay. This ensures keyboard and screenreader users can navigate the menu safely and that focus doesn't bleed through to hidden content.

5

Make timeline responsive on mobile

P2

Collapse the 3-column alternating timeline to a 2-column (dot + content) layout below md breakpoint. The current implementation forces 2×158px content columns on a 360px screen, making bullet text tight. This is the most involved fix but the timeline is the most important section of the portfolio.

6

Remove duplicate email CTA

P2

The email address link and "Email Directly" button both go to the same mailto:. Replace the primary button with an additive action (CV download, or just remove it) so the two CTAs have clear differentiated purpose.

7

Respect prefers-reduced-motion in all animations

P2

The SVG paths already respect this — apply the same logic to per-letter hero entrance, scroll-triggered section reveals, and the timeline scale animations. A single global hook or Framer Motion LazyMotion config covers all cases.

8

Move grain overlay to layout.tsx

P2

Architectural cleanup: the fixed-position grain div is currently inside HeroPaths. Extract it to layout.tsx so its global scope is explicit. This prevents a future regression if the Hero component is ever refactored or replaced.
