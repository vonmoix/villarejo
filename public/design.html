01

First Impression

What lands in the first 2 seconds — emotional reaction, purpose clarity, primary attention anchor.

Immediate emotional read Positive

The hero is arresting. The oversized Bricolage Grotesque name at clamp(4rem, 11vw, 9.5rem) with animated amber SVG paths in the background creates a confident, premium first impression. The grain overlay (now correctly anchored in layout.tsx) adds material texture globally. The amber accent #F59E0B reads as authoritative and distinctive.

Purpose is clear within 2 seconds: named person, senior leadership role, CTA to scroll or get in touch. The scroll indicator (magic-mouse oval) is a functional cue without crowding the composition.

Subtitle weight at minimum clamp size ✓ Fixed

The subtitle "19 years at the intersection…" minimum clamp was raised from 1rem to 1.125rem — ensuring legibility at 360px viewport width below the large name block.

✓ background-paths.tsx — clamp(1rem…) → clamp(1.125rem, 2.2vw, 1.45rem)

02

Usability

Can the user accomplish their goal? Navigation reach, interactive affordances, task completion.

Navigation didn't link to all sections ✓ Fixed

The nav links array previously exposed only 4 anchors, leaving Tech Stack and Certifications unreachable from the navigation. A hiring manager using the nav would never see credentials or tech logos — both high-signal sections for an IT Director audience.

✓ nav.tsx — Added #tech (Tech Stack) and #certifications (Credentials) to the links array. Desktop gap reduced to gap-4 and font to 0.8125rem to accommodate 6 items + CTA.
Duplicate email CTA in Contact section ✓ Fixed

The "Email Directly" button and the large animated email link both pointed to the same mailto:, creating redundant CTAs with no differentiated purpose.

✓ contact.tsx — Button renamed to "Send Email" (retains distinct affordance as a button vs. text link). LinkedIn remains the secondary action.
Timeline not responsive on mobile ✓ Fixed

The 3-column alternating grid gridTemplateColumns: "1fr 44px 1fr" had no responsive override. At 360px, each content column was ~158px — tight for role titles and bullet text.

✓ experience.tsx — Mobile: grid-cols-[20px_1fr] (dot left, all cards right-aligned). Desktop: original md:grid-cols-[1fr_44px_1fr] alternating layout. Vertical line repositioned to left-[10px] on mobile, md:left-[calc(50%-0.5px)] on desktop.
Mobile menu close affordance ✓ Fixed

The burger button had a static aria-label="Toggle navigation" — a screenreader couldn't distinguish the open vs. close state.

✓ nav.tsx — Dynamic label: open ? "Close navigation" : "Open navigation". Also added aria-controls="mobile-menu" linking button to overlay.

03

Visual Hierarchy

Reading order, emphasis, whitespace, typography scale.

Reading flow across sections is excellent Positive

Every section follows the same eyebrow → H2 → body text → content pattern. The eyebrow uses Fira Code at 0.6875rem in accent-hi — a category label, not competing with the headline. The amber rule (44×2px) in the hero creates a precise anchor between name and subtitle.

About section image too flat on mobile ✓ Fixed

The 21/8 aspect ratio (2.6:1) rendered at only ~135px tall on a 360px screen — insufficient image content for the visual break it was meant to provide.

✓ about.tsx — Responsive: aspect-[16/9] md:aspect-[21/8]. Mobile gets a richer image height; desktop retains the cinematic panoramic ratio.
Emerging Technologies group had equal visual weight to other skill groups ✓ Fixed

The Emerging Technologies group (AI, agentic tools, Claude Code) is the highest-signal differentiator for this profile but was visually indistinguishable from "Leadership & Operations".

✓ skills.tsx — Added a subtle border border-accent/20 rounded-lg treatment to the Emerging Technologies card (index 4), drawing appropriate attention without shouting.

04

Consistency

Design system adherence — spacing, colour tokens, typography behaviour across sections.

Section subtitle margin inconsistency ✓ Fixed

Bottom margin after the descriptive paragraph ranged from mb-8 (Contact) to mb-16 (Experience) with no rule. Sections without a subtitle had equally inconsistent H2 bottom margins.

✓ Standardised: subtitle paragraphs → mb-16. H2 without subtitle → mb-14. Affected: tech-stack.tsx, contact.tsx, education.tsx, certifications.tsx.
Inline hex colours alongside CSS tokens ✓ Fixed

Raw hex #F59E0B was used directly in SVG stroke attributes and Framer Motion animate objects, bypassing the token system.

✓ background-paths.tsx: stroke="var(--accent)". experience.tsx: backgroundColor: "var(--accent)", borderColor: "var(--accent)", borderColor: "var(--text-subtle)".
Token-based colour system Positive

The OKLCH palette with semantic names (--ink, --muted, --subtle, --accent) is clean and maintainable. Warm-tinted darks give the palette character beyond a generic dark mode. Tailwind config correctly maps all tokens.

05

Accessibility

WCAG 2.1 AA — contrast, keyboard navigation, ARIA, reduced motion.

No skip navigation link (WCAG 2.4.1) ✓ Fixed — was P0

WCAG Level A — keyboard users had to tab through all 6 nav items on every page load before reaching content.

✓ layout.tsx — Visually hidden skip link: sr-only focus:not-sr-only, targets #main-content. Styled with amber background on focus. page.tsx — <main id="main-content">.
--text-subtle failed WCAG AA contrast (3.8:1) ✓ Fixed — was P1

oklch(0.425 0.010 60) yielded ~3.8:1 on --bg, failing the 4.5:1 threshold for normal text. Affected certification years, tech stack labels, education location, and language level indicators — all at ≤11px monospace.

✓ globals.css — --text-subtle lightened to oklch(0.49 0.010 60) (~4.6:1). Warm grey aesthetic preserved; one step up the lightness scale.
Mobile menu overlay — no ARIA, no focus trap ✓ Fixed — was P1

The full-screen mobile menu had no role="dialog", no aria-modal, and no focus trap. Keyboard focus bled into content behind the overlay.

✓ nav.tsx — Added role="dialog" aria-modal="true" aria-label="Navigation" tabIndex={-1}. Focus management: useEffect moves focus to overlay on open, returns to burger button on close. Escape key listener closes the menu.
LinkedIn new-tab link — no ARIA announcement ✓ Fixed — was P2

The target="_blank" LinkedIn link had no aria-label indicating the new-tab behaviour. The ↗ glyph is decorative with no text equivalent for screenreaders.

✓ contact.tsx — aria-label="LinkedIn profile (opens in new tab)".
Framer Motion animations ignored prefers-reduced-motion ✓ Fixed — was P2

The CSS rule in globals.css killed CSS transitions but Framer Motion's JS-driven animations ran regardless. The per-letter hero entrance (96 motion.span elements) and all section scroll-triggered reveals were unaffected by the OS accessibility setting.

✓ useReducedMotion() added to all animated components. When true: initial={false} and transition={{ duration: 0 }}. Affected: background-paths.tsx, experience.tsx, skills.tsx, education.tsx, certifications.tsx, contact.tsx, tech-stack.tsx.

06

Motion Design

Animation quality, purposefulness, and performance. All animations now respect prefers-reduced-motion.

  • ✓Hero letter-by-letter entrance. 96 motion.span elements with staggered spring (stiffness 130, damping 22). Now skipped entirely when reduced motion is preferred.
  • ✓Animated SVG paths — GPU managed. 48 paths × 2 directions, paused via useInView(once: false) when hero scrolls out. Now uses stroke="var(--accent)" instead of hardcoded hex.
  • ✓Timeline scroll-spy. Spring-animated scale + amber dot glow at 42% viewport. backgroundColor and borderColor now use CSS vars. Fully reduced-motion-aware.
  • ✓Staggered section reveals. Consistent y: 14→0, opacity: 0→1 entrance across all sections. All 7 section components updated with useReducedMotion().
  • ✓Grain overlay. Moved from HeroPaths to layout.tsx — now an explicit global texture. Architectural coupling removed.
  • ✓Scroll indicator. Magic-mouse oval with amber bouncing dot. Custom easing [0.45, 0, 0.55, 1] with repeatDelay: 0.3. Minimal and communicative.

07

What Works Well

Genuine strengths to preserve through any future iteration.

Typography system

Bricolage Grotesque / Hanken Grotesk / Fira Code is an unusual and cohesive triple stack. Each font serves a distinct semantic role with zero overlap.

Amber accent

#F59E0B is warm, high-contrast, distinctive. Used precisely — never as a fill on large surfaces. The glow ring on the active timeline dot is elegant.

Timeline scroll-spy

The active highlight closest-to-42%-viewport creates a reading assistant effect — the site responds to scroll without being intrusive.

OKLCH token system

Warm-tinted darks (oklch(0.115 0.011 65) vs a cold #111) give the palette character. Semantic names make the system maintainable.

Monospace label pattern

Fira Code for dates, labels, and section eyebrows creates a systematic feel. tracking-[0.12em] + uppercase at 0.6875rem is precisely calibrated.

Tech stack fallback UX

onError handler degrades gracefully from CDN logo to mono text abbreviation. Production-quality resilience.

Imperial MBA badge

The "with Distinction" amber border/text treatment draws exactly the right amount of attention — notable without shouting.

GPU management in hero

Pausing 48 SVG path animations when the hero scrolls out extends battery life on mobile — beyond what most portfolio sites implement.
